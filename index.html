Tutorial

Long

				      
							@swim
							def pat():
					    	D("bd")
					    	again(pat)

Short

				      
					      Pa * d("bd")
`swim` indicates that the function is called again regularly, in rhythm
The definition of the function follows the ususal Python syntax.
`D` indicates that the pattern passed as an argument is sent to SuperDirt to be interpreted as a musical pattern
For now our pattern only contains one kick-drum sound, but we will see just now more complex ones.
`again` indicates that once the function is done with its musical duties, it schedules itself to be called again at the next bar
Long 
				      
					@swim
					def pat(p=1, i=0):
					    D("bd sd", i=i)
					again(pat, p=1, i=i+1)
Short 
					  
					    Pa << D("bd sd")
We have now added a snare drum to our pattern, which now spans two beats.
We have to indicate which part of the pattern is played at a particular call of the function.
This is done using an iterator argument, here called `i`, which, as it is implemented here, indicates how many times the function has already been called (starts at zero, is incremented of one at each successive call).
Note that the short syntax does not require that you use an iterator.
Long 
				      
					@swim
					def pat(p=1, i=0):
					    D("bd sd . sd", i=i)
					    again(pat, p=1, i=i+1)
Short 
				      
					Pa << D("bd sd . sd")
The dot indicates a silence in a pattern.
Pitched patterns: 
				      
					@swim
					def pat(p=1, i=0):
					    D("bd sd . sd", i=i)
					    D("synth", midinote="C4 E4 G4 B4", i=i)
					    again(pat, p=1, i=i+1)
This does not have a short syntax equivalent.
We can add another pattern to be played simultaneously.
The main pattern has only one element, which specifies the instrument, and the pitch is indicated by a secondary pattern passed to the argument `midinote`.
We here play an arpeggio over the course of four beats.
The presence of two senders is not translatable in short syntax.
Silences can be put in parameters: 
				      
					@swim
					def pat(p=1, i=0):
					    D("bd sd . sd", i=i)
					    D("synth", midinote="C4 E4 . B4", i=i)
					    again(pat, p=1, i=i+1)
One of the notes can be replaced by a silence.
Chord shorthand: 
				      
					@swim
					def pat(p=1, i=0):
					    D("bd sd . sd", i=i)
					    D("synth", midinote="C4@maj7", i=i)
					    again(pat, p=1, i=i+1)
This pattern can be rewritten with a chord name shorthand. Numerous scales and chords presets are integrated in the language.
Chord inversion: 
				      
					@swim
					def pat(p=1, i=0):
					    D("bd sd . sd", i=i)
					    D("synth", midinote="C4@maj7^2", i=i)
					    again(pat, p=1, i=i+1)
We can make chord inversions.
Polyphonic chord: 
				      
					@swim
					def pat(p=1, i=0):
					    D("bd sd . sd", i=i)
					    D("synth", midinote="{C4@maj7}", i=i)
					    again(pat, p=1, i=i+1)
We can also make chords polyphonic.
Rand, range, ramp and repeat 
				      
					@swim
					def pat(p=1, i=0):
					    D("bd sd . sd", i=i)
					    D("synth",
					        midinote="C4|E4 C4~G4 C4!3",
					        cutoff="[100:1000, 100]",
					        i=i)
					    again(pat, p=1, i=i+1)
The pipe operator allows to flip a coin to chose between C and E.
The tilde operator creates a ramp between two extremal values (here it results in a chromatic ascent from C to G).
The bang/exclamation point operator instructs to repeat the value it is applied to a certain number of times.
The colon and square brackets allow to generate a range of evenly-spaced values, here cutoff values for a low-pass filter going from 100 to 1000 by steps of 100 (100, 200, 300 etc).
p = period of a function 
				    
				      @swim
				      def pat(p=0.5, i=0): # p=0.5: the function goes twice as fast
				          D("bd sd . sd", i=i)
				          D("synth",
				              midinote="C4|E4 C4~G4 C4!3",
				              cutoff="[100:1000, 100]",
				              i=i)
				          again(pat, p=0.5, i=i+1)
r = rate of incrementation 
				      
					@swim
					def pat(p=1, i=0):
					    D("bd sd . sd", i=i)
					    D("synth",
					        midinote="C4|E4 C4~G4 C4!3",
					        cutoff="[100:1000, 100]",
					        i=i)
					    again(pat, p=1, i=i+1)
d = divisor, skips events when i is a multiple 
				      
					@swim
					def pat(p=1, i=0):
					    D("bd sd . sd", i=i)
					    D("synth",
					        midinote="C4|E4 C4~G4 C4!3",
					        cutoff="[100:1000, 100]",
					        i=i)
					    again(pat, p=1, i=i+1)
The argument p controls the period of time after which the functions calls itself back. The lower it is, the faster the function goes.
